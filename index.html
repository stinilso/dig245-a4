Zoom.com Usability Test

Although many of us use Zoom often, there are many capabilities of the website and app that users might not be aware of. We had a user test the usability of Zoom.comâ€™s settings in particular.

1

Go to Zoom.us and login

This step was pretty intuitive and the user had no trouble, since the button was in the upper-right corner where it is expected.

2

Allow all participants to screen share

The user felt anxious and lost on the site for about 30 seconds, but eventually completed the task.

3

Enable a waiting room

The user quickly recalled the location of the setting at the top of the page.

4

Disable the meeting password requirement

The user got frustrated and used Command + F to search for this particular setting.

5

Allow both the host and the participants to record the meeting

This task was simple since the layout was familiar.

Conclusions

Once the user found and changed the initial setting, they were able to complete the other tasks quickly, in one instance with the help of Command + F (the site did not have a search bar). We recommend that the website owners should add a search bar to the settings page and make the settings page better organized, with clearer differentiation between various options.
